====================================================
⚙️ 4. How JVM schedules threads
====================================================

- JVM does NOT schedule threads directly
- OS scheduler handles thread execution
- Java threads map to native OS threads
- Time slicing + preemptive scheduling
- Priority is only a hint


====================================================
🔒 5. ReentrantLock internal working
====================================================

- Built on AbstractQueuedSynchronizer (AQS)
- Uses CAS (Compare-And-Swap)
- If lock free → thread acquires
- If locked → enters FIFO queue
- Uses park/unpark for blocking
- Supports fairness + reentrancy


====================================================
🔄 6. Context Switching
====================================================

Steps:
1. Save current thread state
2. Store in Thread Control Block
3. Load next thread state
4. Resume execution

Cost:
- CPU overhead
- Cache disruption
- Performance impact
